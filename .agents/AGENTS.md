# Project Rules & Behavioral Guidelines

## Gestion des Patchnotes
- **Mise à jour automatique** : Tu dois penser à mettre à jour les patchnotes (les fichiers d'historique de mise à jour) à chaque fois que tu estimes que le site web ou le bot passe à une version supérieure (vX.X.X).
- **Séparation Site / Bot** : Il doit y avoir deux patchnotes distincts :
  1. Un patchnote pour le Site Web.
  2. Un patchnote pour le Bot Discord.
- Ne l'oublie pas à la fin des grosses sessions de modifications.

## Automatisation Git (Add, Commit, Push)
- **Automatisation Git obligatoire** : À la fin de chaque tâche ou session de modifications, tu dois automatiquement effectuer un `git add`, un `git commit` (avec un message explicite), et un `git push` sans demander de confirmation ou d'autorisation supplémentaire. Cette étape doit être systématique.
